class ListenerCondition

Condition contains details for one aspect of the current state of this API Resource.

Properties

  • link
    lastTransitionTime: StringSource

    lastTransitionTime is the last time the condition transitioned from one status to another. This should be when the underlying condition changed. If that is not known, then using the time when the API field changed is acceptable.

  • link
    message: String(length <= 32768)Source

    message is a human readable message indicating details about the transition. This may be an empty string.

  • link
    observedGeneration: Int(isPositive)?Source

    observedGeneration represents the .metadata.generation that the condition was set based upon. For instance, if .metadata.generation is currently 12, but the .status.conditionsx.observedGeneration is 9, the condition is out of date with respect to the current state of the instance.

  • link
    reason: String(length.isBetween(1, 1024), matches(Regex("^[A-Za-z]([A-Za-z0-9_,:]*[A-Za-z0-9_])?$")))Source

    reason contains a programmatic identifier indicating the reason for the condition's last transition. Producers of specific condition types may define expected values and meanings for this field, and whether the values are considered a guaranteed API. The value should be a CamelCase string. This field may not be empty.

  • link
    status: "True"|"False"|"Unknown"Source

    status of the condition, one of True, False, Unknown.

    type: String(length <= 316, matches( Regex( #"^([a-z0-9]([-a-z0-9]*[a-z0-9])?(\.[a-z0-9]([-a-z0-9]*[a-z0-9])?)*/)?(([A-Za-z0-9][-A-Za-z0-9_.]*)?[A-Za-z0-9])$"# ) ))Source
