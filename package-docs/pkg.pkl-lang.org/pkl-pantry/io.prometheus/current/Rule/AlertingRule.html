class AlertingRule

Alerting rules allow you to define alert conditions based on Prometheus expression language expressions and to send notifications about firing alerts to an external service.

Known subtypes:
Known usages:
All versions:

Whenever the alert expression results in one or more vector elements at a given point in time, the alert counts as active for these elements' label sets.

More details: https://prometheus.io/docs/prometheus/latest/configuration/alerting_rules/#rule

Properties

  • link
    alert: StringSource

    The name of the alert. Must be a valid label value.

  • expand_morelink
    expr: StringSource

    The PromQL expression to evaluate.

    Every evaluation cycle this is evaluated at the current time, and all resultant time series become pending/firing alerts.

  • expand_morelink
    `for`: Duration?Source

    Alerts are considered firing once they have been returned for this long.

    Alerts which have not yet fired for long enough are considered pending.

    Default if omitted: 0.s

    Labels to add or overwrite for each alert.
